Pipeline architecture

Four decoupled HTTP steps (writer → makeup → director → Seedance). Each step reads typed JSON from SQLite, calls one LLM or media backend, and writes the next artifact — so you can retry or swap models per stage.

Director

LLM expands raw drama into a timed storyboard, VO script, and cast — the contract for every later step. API step: POST /api/runs/:id/writer.

Input
Plain story text on the run: drama_input.
Output
layer1_output: shots with duration hints, script string, characters, dialogue (~1 min total).
Visual

Planner LLM turns Layer 1 into English still prompts; image gen renders character + scene refs for consistent I2V. API: POST /api/runs/:id/makeup.

Input
layer1_output (characters, storyboard shot ids).
Output
makeup_output: character_image_urls, scene_image_urls (HTTP refs to stills).
Prompt

LLM maps each beat to Seedance segment prompts (camera, motion, duration). API: POST /api/runs/:id/director — consumes layer1_output only.

Input
layer1_output (storyboard + script context).
Output
layer2_output with ordered seedance_prompts (per-segment text for video).
Seedance

Seedance I2V per segment (prompts + makeup stills); ffmpeg merges; upload returns one URL. API: POST /api/runs/:id/seedance (202 async + status poll).

Input
layer2_output and non-empty makeup_output (refs required before merge).
Output
layer3_output.video_url — single merged MP4 for the run.
