Seedance multi-agent pipeline — local run
==========================================

1) Python 3.11+, venv and deps:

   cd /path/to/seedance
   python3.11 -m venv .venv
   .venv/bin/pip install -r backend/requirements.txt

2) Environment (repo root `.env`, read by pydantic-settings):

   Option A — Butterbase AI gateway + Storage (recommended):
   BUTTERBASE_API_URL=https://api.butterbase.ai
   BUTTERBASE_APP_ID=app_your_app_id
   BUTTERBASE_API_KEY=bb_sk_...        # console Service Key: chat + Storage REST
   # If gateway supports JSON mode: BUTTERBASE_JSON_RESPONSE=true

   Seedance / ModelArk (video + makeup images):
   SEEDANCE_2_0_API=...                # BytePlus Ark API Key
   # MAKEUP_IMAGE_MODEL defaults seedream-4-0-250828 (BytePlus ModelArk Seedream 4.0 docs)
   # For inference endpoint, use ep-…
   # Optional: ARK_IMAGE_BASE_URL=...      # default same region as SEEDANCE_ARK_BASE_URL
   # Optional: SEEDANCE_VIDEO_MODEL=...    # default dreamina-seedance-2-0-260128

   Merge + upload:
   - Install **ffmpeg** on PATH, or set FFMPEG_PATH to the binary.
   - Merged mp4 uploads via Butterbase Storage **POST /storage/{app_id}/upload**; default **10MB per file** (adjust in console). Long videos may fail upload; then last Seedance segment URL is returned and `upload_error` is set in meta.

   Option B — Direct OpenAI or other OpenAI-compatible API (LLM only):
   OPENAI_API_KEY=...
   OPENAI_BASE_URL=https://api.openai.com/v1
   LAYER1_MODEL=gpt-4o-mini
   LAYER2_MODEL=gpt-4o-mini
   (Without Butterbase, multi-segment + ffmpeg still run, but **no** Storage upload; `video_url` is the last Seedance temporary URL.)

3) Start API (from repo root so `seedance_video` imports work):

   .venv/bin/uvicorn backend.main:app --reload --host 127.0.0.1 --port 8000

4) Frontend (new terminal):

   cd frontend && npm install && npm run dev

   Open http://127.0.0.1:5173 — Vite proxies `/api` to 8000.

Butterbase deployment
---------------------
- Apply `butterbase/schema/runs_v1.json` to your Butterbase app (MCP or REST); if `runs` already exists, **ALTER TABLE runs ADD COLUMN makeup_output jsonb**.
- Configure RLS and CORS per `butterbase/RLS_AND_CORS.txt`.
- When porting backend orchestration to Butterbase Serverless Functions, keep the same JSON contracts as `contracts/*.schema.json`.
